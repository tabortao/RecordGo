# 2026-01-25 创建任务页 Tab 遮挡修复与美化

## 背景与目标
- 创建任务页（普通创建 / AI 智能创建）当前 Tab 相关的某个 `div`（Element Plus 的 active-bar）出现遮挡下方文字的问题。
- 优化 Tab 切换的交互反馈与配色，使整体更清晰、更精致，并兼容深色模式。

## 范围
- 页面：创建任务页（普通创建、AI 智能创建两个 Tab）
- 重点：Tab Header 与内容区的层叠、间距、active 状态样式、色彩与对比度

## 方案
1. 定位遮挡来源
   - 检查 `el-tabs__active-bar`、`el-tabs__header` 与内容区之间的 margin / padding / position / z-index。
2. 修复遮挡
   - 隐藏默认 active-bar 或将其限制在 header 内部。
   - 明确 header 高度与下边距，确保内容区不被覆盖。
3. 优化 Tab 切换体验与配色
   - 选中态：更明显的背景/描边/文字对比；增加轻微动效（仅 CSS）。
   - 未选中态：弱化但仍可读；hover 更明确。
   - 深色模式：同步调整背景与边框透明度，保证对比度。
4. 验证
   - 视觉：Tab 切换不遮挡内容；下方文字可见；颜色协调；深色模式正常。
   - 工程：typecheck 通过，无新增诊断。

## 验收标准
- 平板/桌面端：Tab Header 不遮挡内容区任何文字或表单项。
- Tab 切换：选中态清晰、动画自然、无闪烁。
- 浅色/深色模式：配色对比合理，边框与背景一致。
- `pnpm run typecheck` 通过，VS Code 诊断无新增错误。
