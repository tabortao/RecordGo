# 美化领取记录与创建心愿

## 目标
- 美化「领取记录」页面，记录中的心愿图标与「心愿」页面展示保持一致（同一套解析与渲染逻辑）。
- 美化「创建心愿」页面，Emoji 选择区域更简洁不拥挤，图标名称更短更易读（如“玩家游戏”→“游戏”）。

## 范围
- 前端页面：领取记录页、创建心愿页、心愿页（仅做图标渲染逻辑复用与一致性调整）。
- 图标渲染：抽出统一的 icon 解析/emoji 解析工具或组件，供心愿页与领取记录页共同使用。
- UI：基于现有 SettingsShell/SettingsCard 风格，统一间距、卡片层级、可点击态与移动端可读性。

## 方案
### 1) 统一心愿图标渲染
- 抽取 `isEmojiIcon / emojiChar / resolveWishIconSrc`（含内置图标、上传图标、后端对象 key/presign）为公共工具或组件。
- 领取记录页不再只渲染 `<img>`：当 icon 为 emoji 时渲染 emoji 字符；图片走统一解析。
- 保留记录页“找不到心愿时按名称回退匹配内置图标”的行为，但回退结果也走统一解析。

### 2) 创建心愿页 UI/UX 优化
- Emoji 区域：改为更疏朗的网格（移动端 6 列/平板 8 列），统一圆角与选中态；避免一行过挤。
- 分类标题：名称简化；同时减少视觉噪音（弱化边框/背景、使用更轻的标签样式）。
- 预览区：强化“已选图标”的展示与更换入口；上传与 emoji 选择分区更清晰。

## 验证
- 手动：心愿页与领取记录页同一个心愿在两处显示的图标一致（emoji/内置/上传三种情况）。
- 手动：创建心愿页在手机宽度下 emoji 不拥挤、可点选、选中态清晰，分类名称更简洁。
- 工程：运行 `pnpm run lint; pnpm run lint:fix; pnpm run typecheck` 无报错。
