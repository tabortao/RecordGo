# 2026-02-05 任务自定义金币与创建页提示调整

## 目标
- 创建任务时支持选择“完成时自定义金币”：任务的金币值在完成时由用户输入决定。
- 用户点击“完成任务/打卡一次”时弹窗选择金币：提供 1-10 快捷选项 + 自定义输入。
- 删除创建任务页的两个提示区块（“提示：重复任务建议…” 与 “两种方式，快速创建…”）。

## 范围与约束
- 前端：Vue 3 + Vite + Element Plus + Pinia；字段与后端保持一致。
- 后端：Gin + GORM + SQLite（纯 Go 驱动）；数据迁移必须使用 ORM（GORM AutoMigrate/迁移逻辑）。

## 实施步骤
1. 后端：为 Task 增加“奖励模式”字段（普通固定金币 / 完成时自定义金币），并执行 GORM 迁移。
2. 后端：扩展完成任务相关接口，允许在完成时携带 `custom_coins`，在奖励模式为“自定义”时以该值结算金币。
3. 前端：创建/编辑任务页增加金币选项：固定金币与“完成时自定义金币”。
4. 前端：完成任务/打卡一次时，如果该任务为“完成时自定义金币”，弹窗选择 1-10 或自定义并提交；撤销完成不需要弹窗。
5. 前端：移除创建任务页两个提示区块对应的 DOM。

## 验证标准
- 创建任务选择“完成时自定义金币”后，列表显示正常，完成时弹窗可选 1-10 或输入自定义并成功结算金币。
- 固定金币任务完成流程不变。
- 重复任务（occurrence complete）同样支持自定义金币（若该任务选择自定义）。
- 前端通过 lint/typecheck；后端可正常编译运行，接口返回结构不变（`{ code, message, data }`）。
