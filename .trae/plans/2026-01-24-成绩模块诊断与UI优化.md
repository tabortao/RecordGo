## 目标
- 修复前端诊断报错：新增成绩页面动态导入失败、成绩详情页类型错误
- 优化新增成绩页面：响应式布局、深色主题适配（手机/平板/电脑）
- 修复与优化荣誉墙相关按钮在浅色模式的可见性与一致性

## 范围
- 前端：Vue 3 + Vite + Element Plus
- 涉及页面：ScoreCreatePage、ScoreDetailPage、ScoresPage、荣誉墙相关页面、小成长页面

## 约束
- 字段与后端接口保持一致，不私自新增/改名接口字段
- 不重启前端开发服务（热更新即可）
- 仅修复与 UI/类型相关问题，不引入破坏性重构

## 执行步骤（消项）
1. 修复 ScoreCreatePage.vue 模板语法错误，确保可正常动态导入与渲染
2. 修复 ScoreDetailPage.vue 的 rank 字段类型与渲染逻辑（与后端字段对齐）
3. 优化 ScoreCreatePage.vue 的布局与主题：自适应断点、暗色模式可读性、表单可用性
4. 修复荣誉墙页面“添加荣誉”按钮浅色模式不可见，以及新增荣誉页“保存”按钮不可见
5. 优化小成长页“荣誉墙”按钮样式：与“生长记录”按钮一致、图标填充、右上角填充气泡
6. 运行 lint 与 typecheck，确保诊断清零（允许保留不影响运行的低优先级提示）

## 验证标准
- 访问新增成绩页不再出现 “Failed to fetch dynamically imported module”
- ScoreDetailPage.vue 不再出现 rank 相关 TypeScript 错误
- 新增成绩页在 360px / 768px / 1280px 宽度下布局正常、按钮可点击、表单无溢出
- 浅色与深色主题下，荣誉墙/新增荣誉页的关键按钮对比度充足、可见且可交互
