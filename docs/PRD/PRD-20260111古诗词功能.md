# 古诗词功能
## 主页面
### 一、 界面布局分析 (Layout Analysis)

1.  **顶部导航区 (Header):**
    *   **标题与搜索:** 采用大标题设计，下方紧跟圆角搜索框，提示语为“输入您想搜索的古诗词和作者等”，方便快速定位内容。
    *   **整体色调:** 顶部采用温暖的柠檬黄渐变背景，营造亲和、活泼的学习氛围。

2.  **核心进度区 (Collection Cards):**
    *   **我的背诵诗卡:** 这是一个成就激励模块。显示解锁进度（2/163张），下方是横向滚动的精美插画卡片。
    *   **视觉状态:** 未解锁的内容（如《静夜思》）呈暗色并带有锁头图标，已解锁的则是彩色手绘插图，增强了收集欲望。

3.  **内容列表区 (Curriculum List):**
    *   **分类标签:** 分为“课内同步”和“课外拓展”，支持学龄段筛选。
    *   **卡片式列表:** 每一项包含：
        *   左侧：古诗意境插画。
        *   中间：诗名、评分（星级）、朝代作者、诗词前两句预览。
        *   下方：两个操作按钮，“去学习”和“背诵挑战”。

### 二、 功能细节分析 (Functional Analysis)

1.  **分级教学系统:** 软件支持根据年级（如学龄前、一年级等）进行课内同步教学。
2.  **成就收集系统:** 将古诗词转化为“诗卡”，通过解锁和背诵来完成收集，利用游戏化心理促进学习。
3.  **双向学习模式:** 提供“去学习”（输入/吸收）和“背诵挑战”（输出/考核）两个入口，闭环学习过程。
4.  **视觉审美教育:** 大量使用精美的中国风手绘插图，在学习诗词的同时提升学生的审美。

## “去学习”页面

### 一、 界面布局分析 (Layout Analysis)

1.  **顶部导航与分类 (Top Navigation):**
    *   **左侧:** 简洁的返回箭头。
    *   **中间:** 四个横向并列的标签页（原文、译文、赏析、趣事），当前选中的是“原文”，以深色加粗显示。
    *   **右侧:** 意见反馈/举报图标。
    *   **设计逻辑:** 这种多维度标签设计，让孩子不仅是死记硬背，还能全方位理解背景。

2.  **核心内容区 (Core Content Area):**
    *   **标题框:** 诗名《江南》被包裹在一个带有虚线圆角边框的淡色底纹区域内，右侧配有小喇叭图标，暗示可以点击听读。
    *   **作者信息:** 标题下方居中显示朝代（[汉]）和出处（汉乐府），字号较小，颜色较淡，突出主次。
    *   **正文排版:**
        *   **拼音对照:** 每个汉字上方都带有清晰的拼音，符合小学低年级的使用习惯。
        *   **行间距:** 字间距和行间距非常开阔，阅读感极佳，防止视觉疲劳。
        *   **互动标注:** 部分词语（如“何田田”、“戏”）下方带有**虚线底纹**，这通常表示点击这些词语会弹出注释或释义。

3.  **视觉插画层 (Visual Illustration):**
    *   **背景融入:** 插画占据了页面下半部分约40%的空间。
    *   **风格:** 清新的国风手绘，采用大片的荷叶、跃动的鲤鱼、远处的轻舟，画面意境与诗句“鱼戏莲叶间”高度吻合。
    *   **构图:** 这种底部插画设计既美化了界面，又起到了“看图识意”的辅助教学作用。

4.  **底部交互层 (Bottom Actions):**
    *   **固定浮动栏:** 底部有一个白色圆角的悬浮条，包含两个大按钮。
    *   **左侧按钮（背诵挑战）:** 线性描边设计，属于二级操作。
    *   **右侧按钮（听讲解）:** 亮黄色实心设计，并带有音频波纹图标，属于高频主导操作，引导孩子先听后背。

### 二、 功能细节分析 (Functional Analysis)

1.  **全维度解析功能:** 通过顶部的“译文、赏析、趣事”，将单一的文本转变为丰富的知识库。
2.  **语音导读系统:** 标题的小喇叭和底部的“听讲解”按钮，提供了听觉学习路径。
3.  **拼音辅助系统:** 实时标注拼音，扫清阅读障碍。
4.  **关键点解析:** 正文中的虚线标注，让学生能快速点击查看难懂词汇的即时解释（点读功能）。
5.  **场景化联想:** 精美的动态感插画帮助学生在脑海中建立视觉图像，辅助记忆。

## "背诵挑战"页面
这是对该App“背诵挑战”页面的深度分析。该页面是整个学习闭环中的“输出与考核”环节，核心功能是**语音识别与背诵检测**。

### 一、 界面布局分析 (Layout Analysis)

1.  **顶部导航 (Top Navigation):**
    *   延续了极简风格，仅保留返回键和页面标题“背诵”，最大程度减少干扰，让用户注意力集中在背诵任务上。

2.  **信息提示区 (Information Display):**
    *   **标题与作者:** 仍然保留诗名、拼音和朝代作者信息，作为背诵的引导基础。
    *   **文本隐藏设计:** 与“去学习”页面最大的不同在于，**正文内容被完全隐藏了**。这符合“背诵”的场景逻辑，强迫用户进行记忆提取。
    *   **背景视觉:** 背景插画依然存在，但进行了**淡化（低饱和度/蒙层）处理**。这种视觉降噪处理能营造出一种“安静、专注”的考场氛围，同时保持了视觉的一致性。

3.  **交互控制区 (Bottom Control Panel):**
    *   **音频反馈:** 页面中下部有一排黄色的小圆点组成**音频波纹（Visualizer）**，这实时反映了用户说话的声音波动，给予用户视觉反馈，确认录音正在进行。
    *   **计时器:** 显示“00:00”，记录背诵所用的时间。
    *   **核心按钮（中心）:** 巨大的黄色圆形按钮，当前显示为“暂停”图标。这是页面的**第一优先级（Primary Action）**按钮，操作极其方便。
    *   **辅助功能（两侧）:** 
        *   **左侧“重背”:** 圆形箭头图标，方便用户在出错时快速重新开始。
        *   **右侧“提示”:** 灯泡图标。这是一个非常人性化的设计，当孩子卡壳时，点击可能提供首句提示或关键词提醒，降低挫败感。


### 二、 功能细节分析 (Functional Analysis)

1.  **智能语音识别 (AI Speech Recognition):** 该页面的底层逻辑一定是AI语音转文字技术。系统在后台将用户的发音与标准文本进行实时比对。
2.  **防错与容错机制:** 通过“重背”功能给予用户纠错机会。
3.  **激励与辅助机制:** “提示”功能充当了助教的角色，确保背诵流程能够进行下去，而不是让孩子停在原地。
4.  **沉浸式体验:** 通过隐藏正文、淡化背景、强化录音反馈，构建了一个完整的“背诵实验室”场景。


### 三、 总结：给AI的生成指令 (AI Prompt Summary)

**【AI 绘图/设计指令】**
> **任务:** 设计一款儿童教育App的“背诵录音”界面。
> **风格:** 极其简洁、现代、专注感、符合儿童审美。
> **界面元素:**
> 1. **顶部:** 只有标题“背诵”和返回按钮。
> 2. **中部:** 页面上方显示诗歌标题和作者。页面中间是大幅度淡化的、朦胧的国风荷花插图，**正文内容完全留白**。
> 3. **底部交互区:** 
>    *   中部有一个显著的、发光的黄色圆形按钮，带有“暂停”图标。
>    *   按钮上方是动态的黄色波纹进度条（音频波形）。
>    *   左侧有一个“重新开始”的循环图标，右侧有一个“灯泡”形状的提示图标。
> **视觉细节:** 背景采用柔和的浅灰色或淡蓝色调，按钮采用高亮黄色，字体清晰，整体给人一种沉静但有反馈的录音环境感。
