设计并实现一个任务创建页面的扩展功能，在现有默认TAB基础上新增"AI智能创建"TAB。该功能需满足以下详细需求：

1. 功能模块：
   - 提供两种输入方式：
     * 文本输入框：用于输入学习计划文本
     * 图片上传组件：支持调用AI视觉模型进行图像识别

2. AI处理要求：
   - 自动识别并提取以下任务要素：
     * 必填字段：任务标题、任务描述、任务分类
     * 可选字段：计划时长、任务金币、重复类型
     * 时间字段：开始日期、截止日期（需智能推算合理时间范围）
   - 支持批量识别：当输入内容包含多个任务需求时，能自动分割并生成多个任务草案

3. 用户交互流程：
   - 识别结果以可编辑的卡片形式展示
   - 提供"创建计划"按钮提交确认
   - 允许用户在提交前手动调整所有识别字段

4. 技术实现要求：
   - 前端：
     * 使用TAB组件管理两种创建方式
     * 实现响应式布局适配不同设备
     * 添加加载状态指示器
   - 后端：
     * 设计专用API接口处理AI识别请求
     * 实现任务数据的验证和存储逻辑
     * 确保图片上传的安全处理

5. 质量标准：
   - 识别准确率不低于85%
   - 页面加载时间控制在2秒内
   - 提供清晰的错误提示和重试机制
   - 遵循现有UI设计规范保持风格统一，需适配黑色主题

6. 测试要求：
   - 覆盖各种输入场景测试（短文本/长文本/清晰图片/模糊图片）
   - 验证多任务识别的正确分割
   - 测试边界情况（空输入、无效图片等）
