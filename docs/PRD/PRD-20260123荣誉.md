# 荣誉模块 PRD（2026-01-23）

## 背景与目标
为孩子建立基础荣誉档案，记录荣誉照片、荣誉名称、颁发机构、颁发日期、备注等指标，并与个人信息（年龄通过生日计算、性别、昵称）联动，形成可回溯的荣誉记录。

## 需求范围
### 入口与导航
1. 在“作业家-小成长”页面记录卡片最上方增加“荣誉”按钮，点击进入“荣誉”页面。
2. “荣誉”页面为独立路由，隐藏底部导航。
3. 支持浅色与深色模式，风格与现有页面一致。

### 荣誉主页
1. 顶部展示孩子昵称、年龄、性别，数据与“我的”页保持一致。
2. 荣誉记录按颁发日期倒序排列。
3. 荣誉卡片展示荣誉照片、荣誉名称、颁发日期与颁发机构（无机构时显示“-”）。
4. 点击荣誉卡片进入荣誉详情页。
5. 空态显示引导文案与“新增荣誉”入口。

### 新增与编辑
1. 支持新增荣誉：荣誉名称、颁发机构、颁发日期、荣誉照片、备注。
2. 必填字段：荣誉名称、颁发日期、荣誉照片。
3. 颁发日期默认当天，格式为 YYYY-MM-DD。
4. 图片上传前前端压缩为 webp，上传失败需提示并允许重试。
5. 支持编辑已创建的荣誉，保存后列表与详情同步更新。

### 荣誉详情
1. 展示完整荣誉信息：照片、名称、颁发机构、颁发日期、备注。
2. 支持删除荣誉，删除后返回列表并刷新。

## 数据字段
### 荣誉记录（Honor）
- id
- user_id
- title
- issuer
- awarded_at（YYYY-MM-DD）
- photo_url
- remark
- created_at / updated_at

## 交互与校验
1. 必填字段为空时给出明确提示。
2. 颁发日期必须为有效日期。
3. 图片上传失败时保留本地选择，允许重新上传。
4. 权限与数据隔离以用户维度为准，子账号仅访问自身数据。

## 接口与存储
1. 所有数据存储于后台数据库（SQLite），前后端字段命名保持一致。
2. 照片存储与 S3 对象存储，URL 存储于数据库。

### API（建议）
- GET /honors
- GET /honors/:id
- POST /honors
- PUT /honors/:id
- DELETE /honors/:id
