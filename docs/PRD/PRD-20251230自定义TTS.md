基于项目中我的-习惯与语言模块的语音设置实现，为RecordGo项目的我的-朗读设置功能进行以下增强开发：

1. 功能需求：
- 新增自定义TTS功能模块，支持接入用户指定的TTS服务接口
- 将朗读设置界面重构为双Tab布局：
  * Tab1：系统默认TTS（保留原有功能）
  * Tab2：自定义TTS接口配置
- 自定义TTS接口需包含以下配置项：
  *配置名称、接口地址（API URL）、API Key、音色ID、语速、语调
  * 测试连接按钮及状态指示
  *保存配置

2. 界面要求：
- 采用与一致的Material Design风格
- 配色方案需符合RecordGo现有视觉规范
- Tab切换动画效果平滑流畅
- 每个Tab内保持合理的间距和视觉层次

3. 技术要求：
- 使用React Hooks实现状态管理
- 自定义TTS配置需持久化存储
- 添加输入验证和错误处理机制
- 实现接口测试功能，验证配置有效性

4. 兼容性要求：
- 保持与现有系统TTS功能的并行运行能力
- 适配不同屏幕尺寸

5. 交付物：
- 完整的朗读设置组件代码
- 更新后的项目文档
- 单元测试用例
- 视觉走查报告
