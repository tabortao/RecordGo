### 1. 首屏加载速度优化 (按需加载 + 预渲染 + 懒加载)

为了提升首屏速度，我针对 **JS** 和 **CSS** 进行了体积缩减，并增加了 **Loading 骨架屏**。

- **JS 瘦身（Tree-shaking）**：
  - 移除了 `main.ts` 中 `Element Plus` 的全局全量注册。
  - 改为利用 `unplugin-vue-components` 和 `unplugin-auto-import` 进行组件和 API 的**按需自动引入**。
  - 在 `App.vue` 中引入 `ElConfigProvider` 以保持国际化（中文）支持，而不必引入整个组件库。
- **CSS 瘦身**：
  - 移除了 `main.ts` 中全量 `element-plus/dist/index.css` 的引入（约 300KB+）。
  - 现在样式会随着组件按需加载。
  - 保留了 `dark/css-vars.css` 以支持暗黑模式，并手动补充了 `main.ts` 中用到的 `ElMessage` 的样式。
- **“预渲染”效果（Loading 骨架）**：
  - 在 `index.html` 的 `<div id="app">` 内部注入了轻量级的 **Loading 动画与关键 CSS**。
  - 在 JS 资源下载和解析完成前，用户不再看到白屏，而是立即看到加载状态，提升感官速度。

### 2. 禁止页面放大

- **全局禁止缩放**：
  - 修改了 `index.html` 的 `<meta name="viewport">` 标签。
  - 增加了 `maximum-scale=1.0, user-scalable=no` 属性。
  - 这将对**心愿、作业家、我的**等所有页面生效，彻底禁用双指缩放功能。

### 修改文件清单

1.  `frontend/index.html`: 更新 viewport meta，添加首屏 Loading 结构。
2.  `frontend/src/main.ts`: 移除全量 Element Plus 引入和样式。
3.  `frontend/src/App.vue`: 添加 `<el-config-provider>` 包裹应用。

现在前端应用更加轻量高效，且符合交互规范。
